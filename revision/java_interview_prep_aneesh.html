Object-Oriented Programming (OOP)

The most important topic for any Java interview. Know all 4 pillars cold.

If they ask only one Java question, it will be about OOP. The 4 pillars — Encapsulation, Inheritance, Polymorphism, Abstraction — must roll off your tongue with a clear definition AND a real example each.

Collections framework

HashMap, ArrayList, LinkedList, Set — know the differences and when to use each.

Exception handling

try-catch-finally, checked vs unchecked, custom exceptions — all covered here.

Advanced Java concepts

Interfaces, abstract classes, generics, Java 8 features. Know these for senior-level questions.

Coding questions

These are actual write-the-code questions. Study the logic, not just the output. Write them out by hand at least once.

When asked to write code in an interview: think out loud, explain what you're doing before you do it, and always mention edge cases even if you don't code them.

How to answer Java questions

Communication strategy — how to structure your answers, what to say when you're stuck, and how to sound confident.

The 3-part answer formula for concept questions
1. Define it in one sentence.
"Polymorphism is the ability of a single interface to represent different underlying forms."

2. Explain how it works technically.
"In Java, this happens through method overriding — a subclass provides its own implementation of a method defined in the parent class. At runtime, Java calls the subclass version even if the reference type is the parent."

3. Give a real example.
"For example, if you have an Animal class with a speak() method, and Dog and Cat extend Animal and override speak(), you can have a list of Animal objects and each one calls its own version of speak(). The code doesn't need to know whether it's a dog or cat — it just calls speak()."
What to say when you don't know the answer
Never go silent. Never say "I don't know" and stop. Always bridge.

Template: "I haven't worked with [X] extensively, but from what I understand, [related concept you know]. I'd approach it by [logical reasoning]. I'd also want to [look it up / test it] to confirm."

Example: "I haven't used Java's ForkJoinPool directly, but I know it's related to parallel processing and the work-stealing algorithm. I'd start by understanding the thread pool size relative to available CPU cores, similar to how I think about parallelism in Python's multiprocessing."
What to say before writing code
Always verbalize your plan before you start writing. It shows structured thinking.

Say: "Before I write the code, let me think through the approach. I'm going to [do X] first, then [do Y]. I'll need to handle the edge case where [Z]. Does that approach make sense?"

This buys you thinking time AND impresses them with your communication.
How to connect Java to your data analyst work
If they ask why Java matters for a data analyst role, say:

"Java is often used in enterprise data pipelines — tools like Apache Spark, Hadoop, and Kafka are JVM-based. Understanding Java means I can work closer to the data infrastructure layer, not just the analysis layer. For example, if a Spark job written in Java is producing unexpected results, I can trace through the logic rather than just consuming the output."
5 answers to memorize word-for-word
These come up in almost every Java interview. Know them cold:

1. "Java is platform-independent because Java code compiles to bytecode, which runs on the JVM. The JVM is available on any OS, so the same .class file runs anywhere — write once, run anywhere."

2. "The four pillars of OOP are Encapsulation, Inheritance, Polymorphism, and Abstraction."

3. "== checks reference equality — whether two variables point to the same object in memory. .equals() checks value equality — whether the contents are the same."

4. "A checked exception must be declared or handled at compile time. An unchecked exception — like NullPointerException — occurs at runtime and doesn't need to be declared."

5. "ArrayList uses a dynamic array and gives O(1) random access. LinkedList uses a doubly-linked list and gives O(1) insertion at both ends but O(n) random access."
